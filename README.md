# LeetCode Mini Projects

A collection of small, focused LeetCode problems treated like mini projects.

Instead of just solving problems, each one is:
- Structured
- Cleanly implemented
- Documented with complexity analysis
- Connected to a core idea

---

## 📂 Structure

Each folder contains:

- `main.py` → Implementation
- `README.md` → Problem explanation + approach
- Time & space complexity


---

## 🧠 Approach

Every problem follows a simple process:

1. Understand the brute force solution  
2. Identify the bottleneck  
3. Optimize using the right data structure  
4. Analyze complexity  

---

## 🚀 Why This Exists

To build consistency in problem solving.

Algorithms are easier when you see the pattern behind them
